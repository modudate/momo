# 영업 데모 제작 플레이북 (autorun 템플릿)

이 폴더는 **영업 문의 → 데모 제작**을 빠르게 반복하기 위한 마스터 템플릿이다.
새 문의가 들어오면 이 폴더를 통째로 복사해서 시작한다. 디자인 시스템(OMD)과
Next.js 스캐폴드가 이미 깔려 있으므로 **재설치·재설명이 필요 없다.**

---

## 새 데모 시작하는 법 (사람이 하는 일)

1. `autorun` 폴더를 복사 → 클라이언트 이름으로 새 폴더명 지정 (예: `acme`)
2. 영업 브리프(목적/요건)를 `1.md`에 붙여넣기
3. Claude에게: **"1.md 읽고 데모 만들어줘"**

그 다음은 아래 규칙대로 Claude가 진행한다.

---

## Claude가 지켜야 할 규칙

### 0. 시작 시
- 항상 `1.md`를 먼저 읽고 **목적 / 핵심 기능 / 타깃 사용자 / 데이터 유무**를 한 문단으로 요약한 뒤 착수한다.
- 불명확한 핵심 1~2개만 질문하고, 나머지는 합리적 기본값으로 진행한다 (영업 데모이므로 속도 우선).

### 1. 스택 (고정)
- **Next.js 15 (App Router) + React 19 +  TypeScript 
- 경로 별칭 `@/*` (jsconfig.json).
- 외부 이미지는 `<img>` + Unsplash/CDN URL, `next.config`에서 `images.unoptimized: true`.
- 한글 폰트 Pretendard (이미 layout.jsx에 CDN 링크 포함).
- 상태는 가벼우면 React state, 데모 영속성이 필요하면 `lib/store.js`에 localStorage 기반으로.

### 2. 디자인 (OMD 사용 — 재설치 금지)
- 디자인 시스템은 **이미 설치돼 있다** (`.claude/`, `.agents/skills/`). 절대 다시 깔지 않는다.
- 새 브랜드 톤이 필요하면 `omd:init`으로 이 프로젝트에 맞는 `DESIGN.md`를 부트스트랩한다.
- 전체 화면/랜딩을 처음부터 만들 땐 `omd:harness`, 부분 수정은 `omd:apply`.
- `app/globals.css`의 토큰(색/라운드/간격)을 기준으로 쓰고, 임의 하드코딩 색상은 피한다.
- 기본 베이스는 Toss 계열 뉴트럴 팔레트. 클라이언트 업종에 맞게 primary만 바꾸는 식으로 빠르게.

### 3. 데모 품질 기준
- 모든 화면은 5가지 상태를 고려: **빈 상태 / 로딩 / 에러 / 성공 / 스켈레톤**.
- 목업 데이터는 그럴싸하게 (실제 한국 회사/제품/숫자 느낌). lorem ipsum 금지.
- 모바일 반응형 기본 포함.
- "영업용 데모"이므로 **첫인상(히어로)·핵심 플로우 1개**를 완성도 높게. 나머지는 구색.

### 4. 배포 (Claude가 자동으로)
- 데모 제작이 끝나면 **Claude가 직접 배포까지 실행한다.** 사람이 배포.bat을 누를 필요 없음.
  - 순서: `npm install` (node_modules 없으면) → `npm run build` (에러 확인) → `npx vercel deploy --prod --yes`
  - 배포 성공 후 **프로덕션 URL을 사용자에게 알려준다.**
- 전제: Vercel CLI 로그인은 최초 1회 사람이 `npx vercel login`으로 해둬야 한다 (대화형이라 Claude가 못 함). 로그인돼 있으면 `--yes`로 비대화형 배포가 된다.
- 로그인 안 돼 있거나 배포 실패하면, 그 사실과 해결법(`npx vercel login`)을 알리고 빌드까지만 완료해 둔다.
- 수동 대안: `배포.bat` 더블클릭. 로컬 확인: `npm run dev` → http://localhost:3000

### 5. 견적/기간 문의가 섞여 있으면
- `1.md`에 비용·기간 질문이 있으면 (예: filter B2B 사례), 데모와 별개로
  `견적.md`에 **개발 범위 / 예상 기간 / 비용 레인지**를 간단 표로 정리해 함께 제공한다.

### 6. 문서를 PDF로 만들 때 (계획서·제안서·인력투입표 등)
계약 부속 문서나 계획서·제안서를 **PDF로 만들어 달라**는 요청이 오면, 아래 디자인 템플릿으로 조판해서 출력한다. (워드/한글로 받은 docx도 텍스트만 뽑아 이 디자인으로 재조판)

**제작 방식 (Windows)**
- HTML로 조판 → 스크래치패드에 저장 → **Edge 헤드리스로 PDF 변환**.
- 변환 명령(브라우저 머리말/꼬리말 반드시 제거):
  `msedge.exe --headless=new --disable-gpu --no-pdf-header-footer --user-data-dir=<temp> --virtual-time-budget=9000 --print-to-pdf="<out.pdf>" "file:///<html>"`
  - `--no-pdf-header-footer` 빼먹으면 날짜·파일경로·제목이 자동으로 찍히니 **항상 포함**.
- HTML은 스크래치패드, **완성 PDF는 프로젝트 폴더**에 저장.

**디자인 규칙 (고정)**
- 폰트: **Pretendard** CDN `@import` + `"Malgun Gothic"` 폴백. 본문 글자 크기는 넉넉히(본문 11px 안팎).
- **단일 브랜드 컬러 1개**를 CSS 변수 `--brand`로 두고 전 문서를 그 색으로 통일한다(+ 연한 틴트 `--brand-soft/--brand-mid`, 본문은 중립 다크그레이/검정). 클라이언트 색을 모르면 기본값 `#025afc`(또는 `#6498fe`)를 쓴다.
- **표지(1쪽)**: 흰 배경 + 상·하단 컬러 바 + 한쪽 워터마크(둥근 사각) + 가운데 **로고 자리** → 브랜드 라벨(자간 넓게) → 제목(일부만 `--brand` 강조) → 짧은 영문 부제 → 짧은 밑줄.
- **본문**: 상단 러닝헤더(문서명 + 밑줄) → `h2` 섹션 제목(좌측 컬러 바 + 번호 칩) → 내용은 **카드 / 표 / 콜아웃**으로. 표는 헤더행 `--brand` 배경·흰 글자, 라벨칸은 연한 틴트.
- **페이지 나눔**: 카드에 `overflow:hidden`·`page-break-inside:avoid`를 통으로 걸지 말 것(빈 페이지 생김). 대신 **항목(`.it`) 단위로만** 안 쪼개지게 하고, 섹션 제목은 `page-break-after:avoid`로 다음 블록과 붙인다.

**로고 자리 (보통 비워둔다)**
- 클라이언트 로고가 없으면 **자리만 잡고 비워 둔다.** 표지에 아래 플레이스홀더를 넣어 위치·크기만 확보:
  ```html
  <!-- 로고 자리: 실제 로고 SVG/PNG 받으면 이 안에 교체. 없으면 빈 채로 둔다 -->
  <div class="logo-slot" style="width:216px;height:80px;margin:0 auto 30px;"></div>
  ```
- 실제 로고(svg/png)를 받으면 이 자리에 그대로 넣고, 색은 문서 톤(`--brand`)에 맞출지 원본색 유지할지 사용자에게 확인한다.

**원문 보존**
- 본문 **문구는 사용자가 준 텍스트 그대로** 옮긴다. 디자인 요소(로고·표·타임라인 시각화) 외에 **설명 문구를 임의로 추가하지 않는다.** 표의 합계 등 숫자는 바꾸면 **전부 재계산**해서 맞춘다.

## 폴더 구조

```
autorun/                 ← 이 템플릿 (복사해서 사용)
├─ CLAUDE.md             ← 이 파일 (지시서, 자동 로드)
├─ 1.md                  ← 영업 브리프 (복사 후 교체)
├─ package.json          ← Next 15 + React 19
├─ app/
│  ├─ layout.jsx         ← Pretendard 폰트 포함 루트
│  ├─ globals.css        ← 디자인 토큰 베이스
│  └─ page.jsx           ← 시작 플레이스홀더
├─ 배포.bat              ← Vercel 배포 원클릭
├─ .claude/ .agents/     ← OMD 디자인 시스템 (재설치 금지)
└─ jsconfig / next.config / .gitignore
```

## 안 해도 되는 일 (반복 제거 체크리스트)
- ❌ 디자인 시스템 설치 — 이미 됨
- ❌ Next.js 보일러플레이트 생성 — 이미 됨
- ❌ 폰트/토큰 세팅 — 이미 됨
- ❌ 스택·컨벤션 재설명 — 이 파일이 대신함
